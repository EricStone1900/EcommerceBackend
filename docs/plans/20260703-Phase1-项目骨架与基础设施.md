# Phase 1 开发计划 — 项目骨架与本地基础设施

## 背景

项目目前处于 Phase 0（完全空白，无任何 Go 代码/配置文件/构建文件）。本阶段基于 `docs/specs/01-项目骨架与本地基础设施.md` 构建整个项目的地基，建立 Clean Architecture 四层骨架，跑通本地基础设施和 health check。

### 任务 2：配置加载（Viper）

**文件：** `internal/infrastructure/config/config.go`
- `Config` struct：包含 App、Database、Redis、JWT、Storage 子结构体
- `LoadConfig(path string) (*Config, error)`：Viper 读文件 + 环境变量（env 优先级高于文件）
- 配置项见 `configs/config.example.yaml`

**文件：** `configs/config.example.yaml`（配置模板，占位符而非真实值）
- `server.port`、`server.env`（dev/prod）
- `database.host/port/user/password/dbname/sslmode`、连接池参数
- `redis.url`、`redis.password`、`redis.db`
- `jwt.secret`、`jwt.access_expire`、`jwt.refresh_expire`
- `storage.driver`（local/s3）、S3 系列

### 任务 3：日志初始化（zap）

**文件：** `internal/infrastructure/config/config.go`（已在同一文件，或单独文件 `internal/infrastructure/log/`）
- `InitLogger(env string) (*zap.Logger, error)`
- `env=production` → JSON 格式
- `env=development` → console 格式（易于本地阅读）
- 保存全局 logger 引用，提供 `Sync()` 清理

### 任务 4：GORM 数据库连接

**文件：** `internal/infrastructure/persistence/gorm/gorm.go`
- `NewDB(cfg *config.Config) (*gorm.DB, error)`
- 连接池：`MaxOpenConns=100`, `MaxIdleConns=10`, `ConnMaxLifetime=1h`
- 提供 `HealthCheck(db *gorm.DB) error`（ping 底层 sql.DB）
- 测试：`internal/infrastructure/persistence/gorm/gorm_test.go`

### 任务 5：Redis 连接

**文件：** `internal/infrastructure/cache/redis/redis.go`
- `NewRedis(cfg *config.Config) (*redis.Client, error)`
- 提供 `HealthCheck(client *redis.Client) error`（ping）
- 测试：`internal/infrastructure/cache/redis/redis_test.go`

### 任务 6：依赖注入容器

**文件：** `internal/container/container.go`
```go
type Container struct {
    Config *config.Config
    Logger *zap.Logger
    DB     *gorm.DB
    Redis  *redis.Client
}
```
- `NewContainer(cfgPath string) (*Container, error)` — 依次初始化 Config → Logger → DB → Redis
- `(c *Container) Close() error` — 优雅关闭（DB 连接释放、Redis 关闭、logger Sync）

### 任务 7：Gin 路由骨架

**文件：** `internal/interface/http/router/router.go`
```go
type Router struct {
    engine *gin.Engine
}
func NewRouter(logger *zap.Logger) *Router
func (r *Router) Public(method, path string, handler gin.HandlerFunc)
func (r *Router) Protected(method, path string, handler gin.HandlerFunc, roles ...string)
func (r *Router) Engine() *gin.Engine
```
- 内部实现：Public 和 Protected 当前都注册到 engine（占位），RBAC 中间件下个阶段实现
- 先在 Protected 里加一个日志中间件表明"此处需鉴权"

**路由注册模式（规范）：**
- 所有路由必须在 router 层通过 Public/Protected 注册
- handler 内部不做鉴权判断

### 任务 8：Health Check 接口 + main.go

## 技术决策

| 决策点 | 选择 | 理由 |
|---|---|---|
| CLI 子命令 | `os.Args` 简单解析（不用 cobra）| Phase 1 只有 2 个子命令，无需引入额外依赖 |
| DI 容器 | 手写 container.go（不用 wire/uber-fx）| 符合 spec 要求，代码量小 |
| 域名 | `github.com/EricStone1900/ecommerce-backend` | go.mod 使用此 module path |
| 测试框架 | `testing` + `testify/assert` | 标准 Go 测试 + 断言增强 |
| 日志库 | `go.uber.org/zap` | 结构化日志首选 |
| 配置文件路径 | 硬编码 `configs/config.local.yaml`（可被 `APP_CONFIG_PATH` 环境变量覆盖）| 简化启动 |

## 验证方式

### 编译器验证
```bash
go build ./...
go vet ./...
```

### 测试验证
```bash
# 单元测试（不需要 docker）
go test ./internal/... -v -count=1

# 集成测试（需要 docker-compose up 后）
docker-compose -f deployments/docker-compose.yml up -d
go test ./... -v -count=1 -tags=integration
```

### 运行验证
```bash
# 1. 启动依赖
docker-compose -f deployments/docker-compose.yml up -d

# 2. 运行迁移
go run cmd/server/main.go migrate

# 3. 启动服务器
go run cmd/server/main.go serve

# 4. 验证 health check
curl http://localhost:8080/health
# 期望输出：
# {"status":"ok","database":"connected","redis":"connected"}
```

### 架构约束验证
```bash
grep -r "gorm.io\|go-redis\|aws-sdk" internal/domain internal/usecase
# 必须为空
```
