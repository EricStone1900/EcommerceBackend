# Phase 2 开发计划 — 用户认证与权限 RBAC

## 上下文

Phase 1 已完成：Clean Architecture 四层骨架（domain/usecase/interface/infrastructure）、配置加载（Viper）、日志（zap）、GORM + PostgreSQL 连接、Redis 连接、依赖注入容器、`/health` 接口、docker-compose 本地环境。本阶段在此基础上实现用户注册登录和 JWT 鉴权 + RBAC 权限体系。

**基于 `docs/specs/02-用户认证与权限RBAC.md` 开发。**

---

## 实施顺序（严格按此顺序，前置依赖先完成）

### 任务 1：领域层 + 公共工具

**文件创建：**
- `pkg/response/response.go` — 统一 JSON 响应格式 `{"code":0,"data":{...},"message":"success"}`
  - `Success(data interface{}) Response` / `Error(code int, message string) Response`
- `pkg/errors/errors.go` — 业务错误码 + HTTP 状态映射（Error sentinel 结构体，`errors.Is()` 兼容）
  - 错误码：InvalidRequest(1001), ValidationError(1002), Unauthorized(2001), TokenExpired(2002), Forbidden(2003), UserNotFound(3001), EmailAlreadyExists(3002), InvalidCredentials(3003), InternalError(5001)
- `internal/domain/entity/user.go` — `User` 实体（id, email, password_hash, role, created_at, updated_at）
  - `Role` 枚举：`admin` / `member` / `customer`
  - `DeletedAt` 不放在 domain entity 中（保持纯净），只在 GORM model 中处理软删除
- `internal/domain/port/user_repository.go` — `UserRepository` 接口
  - `CreateUser(ctx, user) error`
  - `GetUserByEmail(ctx, email) (*User, error)`
  - `GetUserByID(ctx, id) (*User, error)`
- `internal/domain/port/token_store.go` — `TokenStore` 接口（refresh token 持久化存储抽象）
  - `SaveRefreshToken(ctx, userID, tokenID, expiration) error`
  - `ValidateRefreshToken(ctx, userID, tokenID) (bool, error)`
  - `DeleteRefreshToken(ctx, userID, tokenID) error`

### 任务 2：JWT 工具

**文件创建：**
- `pkg/jwt/jwt.go` — JWT 令牌生成/验证
  - `Claims` struct：UserID, Role, 标准 RegisteredClaims
  - `NewJWTService(secret, accessExpire, refreshExpire)`
  - `GenerateAccessToken(userID, role) (string, error)`
  - `GenerateRefreshToken(userID, role) (token, tokenID, error)` — 含 jti (UUID)
  - `ValidateToken(token) (*Claims, error)` — 返回业务错误

**依赖：** `github.com/golang-jwt/jwt/v5`, `github.com/google/uuid`

### 任务 3：基础设施实现

**文件创建：**
- `internal/infrastructure/persistence/gorm/user_repo.go` — GORM UserRepository 实现
  - 内部 `userModel` struct（含 `gorm.DeletedAt`）+ `toEntity()` / `toModel()` 转换
  - `TableName()` 返回 `"users"`，GORM 自动过滤软删除
- `internal/infrastructure/cache/redis/token_store.go` — Redis TokenStore 实现
  - Key 格式：`refresh_token:{userID}:{tokenID}`
  - 设置 TTL 匹配 refresh token 有效期

### 任务 4：业务用例（usecase/auth）

**文件创建：**
- `internal/usecase/auth/auth.go` — 共享类型 + `AuthUseCase` struct
  - `RegisterRequest{Email, Password}`, `LoginRequest`, `RefreshRequest{RefreshToken}`
  - `AuthResponse{AccessToken, RefreshToken, User}`, `UserInfo{ID, Email, Role}`
  - `AuthUseCase` — 依赖 `UserRepository`, `TokenStore`, `JWTService`, `Logger`, expires
- `internal/usecase/auth/register.go`
  - 流程：验证邮箱格式 → 密码强度（≥8位，含字母+数字）→ 唯一性检查 → bcrypt 哈希(cost=12) → 存储 → 签发 token → 返回
- `internal/usecase/auth/login.go`
  - 流程：查用户 → bcrypt.CompareHashAndPassword → 签发 token → 返回
  - 密码错误/用户不存在统一返回 `ErrInvalidCredentials`（防止邮箱枚举）
- `internal/usecase/auth/logout.go`
  - 流程：验证 refresh token → 从 Redis 删除 → 返回成功（幂等）
- `internal/usecase/auth/refresh.go`
  - 流程：验证 refresh token → 校验 Redis 中存在 → 删除旧 token（轮换）→ 签发新 token → 存 Redis → 返回
  - 拒绝 access token 用作 refresh token（校验 type claim）

### 任务 5：HTTP 中间件

**文件创建：**
- `internal/interface/http/middleware/auth.go` — JWT 鉴权中间件
  - 提取 `Authorization: Bearer <token>` → 校验 → 将 userID (uint) 和 role 存入 Gin 上下文
  - 提供 `GetUserID(c *gin.Context) uint` / `GetRole(c *gin.Context) entity.Role` 辅助函数
  - 失败返回 `ErrUnauthorized`(401) 或 `ErrTokenExpired`(401)
  - 拒绝 refresh token 用于鉴权（校验 type claim）
- `internal/interface/http/middleware/rbac.go` — 角色校验中间件
  - `RBACMiddleware(allowedRoles ...entity.Role)` — 从上下文取 role → 检查是否在允许列表
  - 空角色列表 = 允许所有已认证用户（AnyRole）
  - 不在列表中返回 `ErrForbidden`(403)

### 任务 6：HTTP Handler + 路由注册

**文件创建：**
- `internal/interface/http/handler/auth.go` — 5 个 Gin handler 方法
  - `Register`, `Login`, `Logout`, `Refresh`, `Me`
  - 每个 handler：JSON 绑定 → 调用 usecase → `response.Success()` 或 `response.Error()`
  - 错误通过 `errors.As()` 识别 `*Error` 类型，提取 HTTP status code

**文件修改：**
- `internal/interface/http/router/router.go`
  - 添加 `authMiddleware gin.HandlerFunc` 字段
  - 添加 `SetAuthMiddleware(mw gin.HandlerFunc)` 方法
  - 更新 `Protected()` 自动拼接 `authMiddleware + RBACMiddleware(roles...) + handler`
  - `roles` 类型改为 `entity.Role`

### 任务 7：容器组装 + main.go

**文件修改：**
- `internal/container/container.go`
  - 添加字段：UserRepo, TokenStore, JWTService, AuthUseCase, AuthHandler, AuthMiddleware
  - `NewContainer()` 中按序初始化新依赖
- `cmd/server/main.go` — 注册路由
  ```go
  // 在 router.NewRouter 后，注册路由前：
  r.SetAuthMiddleware(container.AuthMiddleware)

  // 公开
  r.Public("POST", "/api/v1/auth/register", container.AuthHandler.Register)
  r.Public("POST", "/api/v1/auth/login", container.AuthHandler.Login)

  // 受保护（AnyRole = 空角色列表）
  r.Protected("POST", "/api/v1/auth/logout", container.AuthHandler.Logout)
  r.Protected("POST", "/api/v1/auth/refresh", container.AuthHandler.Refresh)
  r.Protected("GET", "/api/v1/auth/me", container.AuthHandler.Me)
  ```
- `go.mod` — 添加 `golang.org/x/crypto`(direct)、`github.com/golang-jwt/jwt/v5`、`github.com/google/uuid`

### 任务 8：迁移脚本

## 技术选型

| 决策点 | 选择 | 理由 |
|---|---|---|
| JWT 库 | `github.com/golang-jwt/jwt/v5` | 事实标准，活跃维护 |
| 密码哈希 | bcrypt cost=12 | 安全/性能平衡 |
| Refresh token 存储 | Redis（通过 TokenStore 接口） | 支持主动吊销 + 自动过期 |
| Refresh token 轮换 | 每次 refresh 删除旧 token 颁发新 token | 限制被盗窗口 |
| 实体软删除 | GORM model 层处理，domain entity 不暴露 | 保持 domain 纯净 |
| 错误处理 | sentinel `*Error` 带 HTTP code 映射 | usecase → handler 清晰传播 |
| 测试 mock | `testify/mock` | 与 Phase 1 一致 |
