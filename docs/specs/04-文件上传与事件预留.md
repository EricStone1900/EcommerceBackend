# 阶段 4 / 7 — 文件上传与微服务预留点（事件机制）

## 上文衔接

阶段 1-3 已完成：分层骨架、用户认证与 RBAC、商品管理 API。本阶段是整个系统里**承上启下最关键的一步**——要实现文件上传功能，并且要把"未来接入 OCR 微服务"这个预留点用真正的事件机制做出来，而不是简单留个 TODO 注释。请继续在已有目录结构里开发。

## 全局架构约束（必须遵守，本阶段尤其重要）

- `domain/port` 下要定义 `Storage` 接口（不关心具体存在本地还是云上）和 `EventPublisher` 接口（不关心未来订阅者是进程内代码还是真实的远程微服务）
- 这两个接口的方法签名都要带 `context.Context` 参数，返回 `error`，体现"这是一次可能失败、未来可能是远程调用"的设计意图
- usecase 层只依赖这两个接口，不知道也不关心 Storage 具体是磁盘还是 S3、EventPublisher 具体是进程内 channel 还是真实的消息队列

## 本阶段目标

1. 实现图片/文档/视频上传接口（本地磁盘存储）
2. 上传成功后发布一个 `file.uploaded` 事件（本阶段事件订阅者只需要做一件简单的事，比如打日志或更新文件状态，**不需要实现真正的 OCR 识别**，目的是验证整个事件链路是通的，未来 OCR 微服务接入时只需要新增一个真正的订阅者，不需要改这里的任何代码）

## 具体任务清单

1. `domain/entity` 下定义 `File` 实体：id、owner_id（上传者）、type（image/document/video）、original_name、url、size、status（如 `pending`/`processed`）、created_at
2. `domain/port` 下定义：
   - `Storage` 接口：`Upload(ctx, fileBytes, filename string) (url string, err error)`、`Delete(ctx, url string) error`
   - `EventPublisher` 接口：`Publish(ctx, eventName string, payload any) error`、`Subscribe(eventName string, handler func(ctx context.Context, payload any) error)`（如果你觉得用真实消息队列客户端的 Subscribe 模式不一样，可以调整接口设计，但要保证 usecase 层调用 Publish 时完全不知道底层实现）
3. `infrastructure/storage/local` 下实现本地磁盘版本的 `Storage`（写入到容器内某个挂载目录，本地用 docker-compose 的 volume 持久化，方便重启不丢文件）
4. `infrastructure/event` 下实现一个简单的进程内事件总线（用 Go channel 或者一个轻量的 pub/sub 库都可以），作为 `EventPublisher` 接口的本阶段实现。在这个实现里，明确写注释说明"未来替换为 NATS/RabbitMQ 等真实消息队列后，订阅者可以是独立的远程微服务进程，这里的接口不需要变"
5. `usecase/upload` 下实现上传用例：接收文件 → 校验类型和大小（图片/文档/视频要有合理的大小上限和类型白名单） → 调用 Storage.Upload → 保存 File 元数据到数据库 → 调用 EventPublisher.Publish 发布 `file.uploaded` 事件（payload 至少包含 file_id、type、url、owner_id）
6. 注册一个事件订阅者（可以放在 `usecase/upload` 初始化时，或者单独的 `usecase/fileprocessing` 模块）：监听 `file.uploaded` 事件，如果类型是 image，模拟"OCR 处理"——可以只是打一条日志说明"此处未来会调用 OCR 微服务"，然后把文件状态从 `pending` 更新为 `processed`。这是为了验证事件订阅、消费、状态回写整条链路都是通的
7. 路由设计：
   ```go
   router.Protected("POST", "/api/v1/upload/image", uploadHandler.UploadImage, AnyRole)
   router.Protected("POST", "/api/v1/upload/document", uploadHandler.UploadDocument, AnyRole)
   router.Protected("POST", "/api/v1/upload/video", uploadHandler.UploadVideo, AnyRole)
   ```
   或者你认为统一一个接口 + type 参数更合理也可以，自行判断并说明理由
8. 预留一个 `assistant`（智能小助手）模块占位接口，比如 `usecase/assistant` 下定义一个 `GenerateProductDescription(ctx, productID) (string, error)` 接口，当前用 mock 实现返回固定文案即可，但接口设计要体现"未来是一次可能超时失败的远程调用"
